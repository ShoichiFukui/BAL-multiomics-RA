# BAL multi-omics (RA vs. sarcoidosis) — Final Analysis Scripts

This directory contains the curated, reproducible R / Python pipeline used in the
the accompanying manuscript on rheumatoid arthritis (RA) BAL multi-omics
profiling versus sarcoidosis controls (n = 35; RA = 24, sarcoidosis = 11).

## Directory layout

```
scripts/
├── README.md                                # this file
├── 00_run_all.R                             # end-to-end runner (sources steps in order)
├── 01_BayesPrism_Deconvolution.R            # scRNA-seq atlas + BayesPrism deconvolution
├── 02_PostDeconvolution.R                   # builds RA_ILD_Workspace.RData
├── 03_Analysis.R                            # master analysis script (sources analysis_modules/)
├── 05_MOFA2_RA_only.R                       # RA-specific MOFA2 (n = 24, 5 views)
├── 06_Comprehensive_Figures.R               # generates ALL main + supplementary figure panels and Supplementary Data 3
├── analysis_modules/                        # sourced by 03_Analysis.R
│   ├── Enhanced_Analysis.R                  # DEG, GSEA, GSVA, effect sizes
│   ├── Infection_Prediction.R              # BAL Th17.1 / OXPHOS infection prediction
│   ├── CT_Multiomics.R                     # CT-quantified ILD progression analysis
│   #     outputs: CT_vs_Multiomics_correlations.csv (Suppl. Data 6, correlations)
│   #              Progression_AllOmics_Comparison.csv (Suppl. Data 7, progressor-vs-stable)
│   └── Integration_Outputs.R                 # final integration outputs
│   # (the 6-view MOFA2 model is built directly in 03_Analysis.R; all main and
│   #  supplementary figure panels are generated by 06_Comprehensive_Figures.R)
├── install_dependencies.R                    # CRAN/Bioconductor/GitHub package install
└── CT_GMM/
    ├── lung_gmm_batch_v3_FINAL.py                       # production batch GMM pipeline (used for the manuscript)
    └── lung_gmm_interactive.ipynb                       # 3D Slicer Jupyter notebook (optional prototype)
```

## Required software

- **R ≥ 4.3** (DESeq2, Seurat v5, Harmony, BayesPrism, fgsea, GSVA, MOFA2,
  pROC, randomForest, tidyverse, patchwork, openxlsx)
- **Python ≥ 3.10** (numpy, scikit-learn, SimpleITK, pydicom, lungmask) — for the
  CT GMM pipeline (`CT_GMM/lung_gmm_batch_v3_FINAL.py`); also python-pptx /
  openpyxl for figure packaging utilities
- **3D Slicer v5 + DensityLungCT GMM Segmentation extension** — optional, only
  required if you want to re-run the interactive prototype notebook
  `CT_GMM/lung_gmm_interactive.ipynb`. The manuscript analyses used
  `lung_gmm_batch_v3_FINAL.py` (pure Python), not Slicer.
- Sufficient memory (~32 GB) for `01_BayesPrism_Deconvolution.R`

### Reproducibility / environment notes
- **CT GMM environment**: `scikit-learn==0.23.2` is pinned to reproduce the exact
  GaussianMixture tissue partition and requires Python 3.8-3.9 (its own env);
  the rest of the pipeline is R and independent of it.
- **BayesPrism**: install a fixed release/commit (e.g. `devtools::install_github("Danko-Lab/BayesPrism@<tag>")`) for exact reproducibility.
- **KEGG**: pathway enrichment fetches KEGG at run time; results reflect the KEGG
  release on the analysis date (stated in Methods). `sessionInfo()` is written to
  `results/session_info.txt` for the full package-version record.

## Execution order

```bash
# 1. Reference atlas + deconvolution (very heavy; one-time)
Rscript scripts/01_BayesPrism_Deconvolution.R

# 2. Build the master workspace
Rscript scripts/02_PostDeconvolution.R

# 3. Main analyses (DEG, GSEA, GSVA, effect sizes, infection, CT progression)
Rscript scripts/03_Analysis.R

# 4. MOFA2 (full-cohort then RA-only)
Rscript scripts/05_MOFA2_RA_only.R

# 5. Generate ALL panels (Fig 1-5, Suppl Fig 1-4) and Supplementary Data 3
Rscript scripts/06_Comprehensive_Figures.R

# 6. CT-quantified ILD progression (production pipeline used for the manuscript)
python scripts/CT_GMM/lung_gmm_batch_v3_FINAL.py
#    Or, optionally, run the prototype notebook in 3D Slicer:
#    CT_GMM/lung_gmm_interactive.ipynb
```

## Manuscript-cited statistical conventions

- All Wilcoxon and Spearman tests use `exact = TRUE`.
- `set.seed(42)` is set immediately before any stochastic step
  (random forest, bootstrap CI, permutation testing).
- Multiple testing correction: Benjamini-Hochberg (FDR) is applied within each
  analysis family (DEG, GSEA, GSVA, microbiome, the RA-vs-sarcoidosis effect-size
  screen, the infection-biomarker screen, and the cytokine/CT-progression screen).
  All `p.adjust()` calls use `method = "BH"`. The correction family (the set of
  tests jointly adjusted) is stated in each Supplementary Table/Data legend.
  Secondary, small-sample analyses (Supplementary Tables 4-5 and the infection
  prediction model) are exploratory and reported with both nominal and
  BH-adjusted P values.
- Cohort: n = 35 (RA = 24, sarcoidosis = 11).

## Cohort definitions

- **RA**: 2010 ACR/EULAR criteria, n = 24
- **Sarcoidosis controls**: granulomatous lung disease comparator, n = 11
- **Healthy serum reference (Saza cohort)**: n = 101; 42 men, 59 women; mean age 58 (SD 9.7)
